# TIL 2026-06-15 - Camping Bot 미션 기능 확장 정리

## 오늘의 목표

캠핑 로봇이 단순히 라이다와 IMU를 보여주는 단계에서 끝나지 않고, 실제 캠핑장 시나리오에 맞게 명령을 받고 움직이는 구조로 확장했다.

중점은 다음 네 가지였다.

- 목적지 기반 미션: 배송, 길안내, 대피, 복귀
- 사용자 확인 기반 장애물 대응
- 로컬 웹 조작 패널
- 엘리베이터 보조 모드

## 현재 로봇 구성

- ROS2: Humble, WSL Ubuntu 22.04
- 로봇 제어 보드: ESP32, IP `192.168.0.10`
- 라이다 보드: ESP32-S3, IP `192.168.0.9`
- 카메라 보드: XIAO ESP32-S3 Sense, IP `192.168.0.11`
- 라이다: LDS14 TRANS V1
- IMU: MPU6050
- 구동: 4륜 구동 + 앞바퀴 아커만 조향
- 부저: ESP32 `GPIO13` 출력

## 오늘 추가한 기능 1: 목적지 기반 미션

`mission_task_manager.py`를 추가해서 사용자가 명령을 보내면 미리 저장된 목적지로 이동할 수 있게 했다.

지원하는 명령:

```text
delivery     배송 위치로 이동
guide        안내 위치로 이동
evacuate     대피 위치로 이동
return_home  시작 위치로 복귀
elevator     엘리베이터 대기 위치로 이동
stop         현재 미션 정지
```

목적지 좌표는 아래 파일에서 관리한다.

```text
software/ros2_ws/src/camping_robot_bringup/config/mission_locations.yaml
```

좌표는 RViz에서 `2D Goal Pose`로 찍거나 `waypoint_recorder`로 기록한 값을 넣으면 된다.

## 오늘 추가한 기능 3: 장애물 보조 흐름

순찰이나 목적지 이동 중 장애물이 오래 지속되면 로봇이 무작정 계속 밀고 가는 것이 아니라, 사용자에게 선택을 요청하는 구조를 추가했다.

상태 확인:

```bash
ros2 topic echo /mission/assistance_request --no-daemon
```

사용자 선택:

```bash
ros2 topic pub /mission/decision std_msgs/msg/String "{data: retry}" --once
ros2 topic pub /mission/decision std_msgs/msg/String "{data: skip}" --once
ros2 topic pub /mission/decision std_msgs/msg/String "{data: stop}" --once
```

의미:

- `retry`: 다시 시도
- `skip`: 현재 목표를 건너뜀
- `stop`: 미션 정지

## 오늘 추가한 기능 4: 엘리베이터 보조 모드

엘리베이터 기능은 아직 로봇이 버튼을 직접 누르거나 문 열림을 자동 인식하는 단계는 아니다.

대신 로봇이 엘리베이터 앞까지 이동하고, 사용자가 웹 패널에서 단계별로 확인해 주는 보조 모드로 만들었다.

시작:

```bash
ros2 topic pub /mission/command std_msgs/msg/String "{data: elevator}" --once
```

상태 확인:

```bash
ros2 topic echo /mission/elevator_status --no-daemon
```

단계:

```text
idle
navigating_to_elevator
waiting_for_elevator
inside_elevator
riding
exited
blocked
```

사용자 확인 명령:

```bash
ros2 topic pub /mission/elevator_decision std_msgs/msg/String "{data: call}" --once
ros2 topic pub /mission/elevator_decision std_msgs/msg/String "{data: entered}" --once
ros2 topic pub /mission/elevator_decision std_msgs/msg/String "{data: floor_selected}" --once
ros2 topic pub /mission/elevator_decision std_msgs/msg/String "{data: exited}" --once
ros2 topic pub /mission/elevator_decision std_msgs/msg/String "{data: complete}" --once
ros2 topic pub /mission/elevator_decision std_msgs/msg/String "{data: cancel}" --once
```

## 오늘 배운 것

### 1. ROS2 토픽이 보인다고 실제 로봇이 움직이는 것은 아니다

`/cmd_vel_raw`, `/cmd_vel`, `/cmd_vel_executed`가 모두 나와도 실제 ESP32가 명령을 못 받으면 로봇은 움직이지 않는다.

그래서 확인 순서는 이렇게 보는 것이 좋다.

```text
명령 토픽 발생
-> safety node가 막지 않는지 확인
-> ackermann_udp_bridge가 UDP를 보내는지 확인
-> ESP32 시리얼에서 cmd_count가 증가하는지 확인
-> 실제 모터가 도는지 확인
```

### 2. 전원과 접촉 불량은 소프트웨어 문제처럼 보일 수 있다

라이다와 ESP32가 가끔 멈추거나, USB에 꽂았다가 빼면 다시 되는 문제가 있었다.

가능한 원인:

- 5V 전원선 접촉 불량
- 순간 전압 강하
- GND 공통 불안정
- ESP32 부팅 타이밍 문제
- 모터 구동 시 노이즈

즉, ROS2 코드가 정상이어도 하드웨어 전원이 흔들리면 토픽이 멈추거나 로봇이 명령을 못 받을 수 있다.

### 3. RViz의 Map 표시에는 QoS가 중요하다

저장된 맵이 안 보이다가 RViz Map 표시의 `Durability Policy`를 `Transient Local`로 바꾸니 보였다.

쉽게 말하면:

```text
Volatile        지금부터 새로 오는 데이터만 받음
Transient Local 이미 발행된 마지막 지도도 받아옴
```

지도는 한 번 발행되고 계속 반복 발행되지 않을 수 있어서, RViz는 `Transient Local`로 받아야 안정적이다.

### 4. 아커만 조향은 제자리 회전이 안 된다

탱크처럼 좌우 바퀴를 반대로 돌려 제자리 회전하는 방식이 아니기 때문에, 목표 위치와 방향을 맞추려면 주차하듯이 앞뒤로 움직여야 한다.

그래서 단순히 목표점으로 직진하는 알고리즘만으로는 방향 맞추기가 어렵다.

현재는 목표 근처에 도착한 뒤, 작은 반경 안에서 전진과 후진을 섞어 방향을 맞추는 방식으로 개선했다.

## 아직 남은 문제

### 1. 오른쪽 뒤 모터가 뻑뻑함

오른쪽 뒤 모터가 손으로 밀 때 잘 안 돌아가는 문제가 있다.

현재는 어느 정도 주행이 가능하지만, 장기적으로는 다음 중 하나가 필요하다.

- 모터 교체
- 감속기 상태 확인
- 축 정렬 확인
- 바퀴 체결 압력 확인
- 좌우 모터 보정값 추가 조정

### 2. 라이다 전원 접촉 안정화

라이다가 멈추면 `/scan`이 예전 값처럼 보이거나 업데이트가 안 될 수 있다.

추천:

- 점퍼선 대신 단단한 커넥터 사용
- 납땜 또는 터미널 블록 사용
- 5V/GND 선 굵기 보강
- 모터 전원선과 센서 전원선 분리
- 전원 입력부에 커패시터 추가

### 3. 장애물 우회는 아직 완성 단계가 아님

현재는 장애물을 감지하고 멈추거나 사용자 선택을 요청하는 구조다.

진짜 순찰 로봇처럼 자연스럽게 우회하려면 Nav2 costmap과 planner 튜닝이 더 필요하다.

특히 확인할 것:

- 로봇 반경
- inflation radius
- obstacle layer
- lidar scan clearing/marking
- 좁은 통로에서 경로 재계획

### 4. 정확한 odom은 아직 부족함

엔코더가 고장난 상태라 현재 위치 추정은 완벽하지 않다.

가능한 개선 방향:

- 엔코더 수리 또는 교체
- IMU yaw 보정 강화
- 라이다 odometry 재검토
- 카메라 기반 보조 위치 추정
- Nav2 튜닝

## 오늘의 결론

이제 Camping Bot은 단순 수동 조종 로봇에서 벗어나, 캠핑장 순찰과 사용자 요청 미션을 처리할 수 있는 기본 구조를 갖췄다.
